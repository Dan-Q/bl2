Example Two

As well as demonstrating additional techniques such as the use of column (rather than row) menus, fontawesome icons, and the updating of multiple <bl-container>s at once, this presentation contains tips on creating and deploying presentations using BodLanes2.

Touch the 'edit' icon on the menu, to the left, to get started.

Editing presentations

Presentations are folders in the presentations folder (or in the case of this one, the examples folder), e.g. presentations/my-presentation. As a minimum, they should contain an index.html file: this is a standard HTML5 file (although you'll probably want to add a few special elements that you might not yet be familiar with).

Some common special elements are:

<bl-template>
Wraps your template - the 'layout' of your presentation.
<bl-container>
Defines areas of a template into which content can be loaded.
<bl-content>
A container for content blocks to load into the template. Content blocks themselves are regular HTML elements with an id attribute.
<bl-grid>
An abstraction of CSS grids suitable for easy creation.
<bl-menu>
A mechanism for organising links into a horizontal or vertical list.

Tools & Workflow

Optionally run a webserver with: npm start. You can then view your presentation by pointing your web browser at e.g. http://localhost:8080/presentations/my-presentation. Using a webserver means that you can also allow others on your network to see your presentations (simply replace 'localhost' with the IP address of your computer) and use advanced features like Google Analytics.

Alternatively, simply open the file (e.g. presentations/my-presentation/index.html) directly in your web browser. Either way, Google Chrome is probably the best bet for a consistent look-and-feel, and is technically similar to many of the recommended deployment solutions. Chrome also provides screen size simulation via the Device Toolbar, which can help to accurately simulate how a presentation will appear on your real signage.

Edit the file in your favourite text editor e.g. Sublime Text or directly in Chrome itself (debugger > Sources > Filesystem, add your presentation folder to the project: changes can now be made from within the browser itself). Refresh periodically to see your changes: under normal circumstances, you'll carry on seeing the content blocks where you left off.

Store any additional files (custom CSS, images, etc.) in your presentation folder, or put it into the extensions folder if it'll be useful across multiple presentations (consider committing it back to GitHub to share with the world, if it might enjoy wider use!)

Deployment

BodLanes presentations are designed to be able to be deployed either via the web (to mobile, desktop, or kiosk clients anywhere, via a web browser), to kiosks as static offline HTML, or as desktop applications via NW.js.

To begin a deployment, run:

npm run build
npm run compile presentations/my-presentation

Your compiled presentation will be placed in output/presentations/my-presentation.

Deployment to Browsers

Deploy the folder to a web server and point browsers at the presentations/my-presentation/presentations/my-presentation/index.html file. presentations/my-presentation/index.html will redirect, if you prefer.

Browsers can in this case be kiosks or desktops as described elsewhere in this presentation, and this can be a good way to remotely manage content as well as to easily integrate with online services such as Google Analytics.

Deployment to Kiosks

Deploy the folder to the kiosk and point the kiosk browser software at the presentations/my-presentation/presentations/my-presentation/index.html file.

Deployment to Desktops

Install NW.js. In the output/presentations/my-presentation folder (or a deployed) copy of it, run:

nw .

(Alternatively, run e.g. nw output/presentations/my-presentation.)
